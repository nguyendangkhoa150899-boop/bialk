2. KIẾN TRÚC & BẢN ĐỒ FILE
----------------------------------------------------------------
  bialk/
   ├─ BotDoMin/
   │   ├─ index.js   : toàn bộ bot casino (1 file lớn)
   │   ├─ panel.js   : web dashboard (HTTP server thuần Node, 0 thư viện)
   │   ├─ database.json : dữ liệu người chơi + lịch sử (TỰ SINH, gitignore)
   │   └─ package.json
   ├─ BotNoi/
   │   ├─ index.js
   │   ├─ src/handlers/messageHandler.js   : prefix ',' -> đọc chữ
   │   ├─ src/handlers/interactionHandler.js: lệnh /join
   │   ├─ src/handlers/voiceHandler.js      : vào voice, hàng đợi, phát audio
   │   ├─ src/utils/ttsApi.js               : gọi API Vbee + cache audio
   │   ├─ src/utils/textFilter.js           : lọc url/emoji khỏi câu đọc
   │   ├─ src/utils/webhookServer.js  *** CODE CHẾT, không dùng ***
   │   └─ src/utils/ngrokManager.js   *** CODE CHẾT, không dùng ***
   └─ HUONG-DAN-UPDATE.txt  (file này)

  BotDoMin/index.js — điểm chính:
   - dbCache: bản RAM của database.json. Tự lưu mỗi 10 giây + lưu ngay
     khi bot tắt (SIGINT/SIGTERM).
   - Key bắt đầu bằng "_" là metadata, KHÔNG phải người chơi:
       _txHistory, _bcHistory, _minesHistory (lịch sử 15-20 ván)
       _bcChannelId, _txChannelId (kênh dùng lần cuối)
       _savedChannels (danh sách kênh đã lưu cho dashboard)
     Key còn lại = userId -> { points, lastDaily, name }
   - Lớn Nhỏ (txState) & Bầu Cua (bcState): vòng lặp setInterval 1 giây,
     khóa cược ở (targetTime - 5s), mở bát ở targetTime. Có cờ
     isProcessing chống chạy chồng, watchdog 120s, tự hồi phục nếu mất
     message do timeout mạng.
   - Dò Mìn (/domin): game tương tác bằng nút bấm (collector).
   - forcedMines: admin đặt vị trí mìn cho ván /domin kế tiếp (theo user
     hoặc '_any' = người tiếp theo bất kỳ). Đặt từ dashboard.
   - Lệnh slash CÒN LẠI: /domin /sodu /diemdanh /chuyentien (người chơi)
     + /addtien /trutien (admin). Mọi lệnh admin khác đã chuyển lên web.

  BotDoMin/panel.js — web dashboard:
   - HTTP server bằng module 'http' built-in (CỐ Ý không dùng express
     để khỏi cài thêm gì trên VPS).
   - Đăng nhập bằng mật khẩu -> cấp token (lưu trong RAM).
   - Chức năng: ép kết quả Lớn Nhỏ/Bầu Cua, ép vị trí mìn, bật/tắt bàn,
     xóa chat bot, quản lý điểm (set/cộng/set tất cả), kênh đã lưu,
     xem lịch sử, trạng thái bot trực tiếp (online/mất kết nối).

  BotNoi — điểm chính:
   - Prefix đọc chữ là dấu PHẨY ',' (vd: ,xin chào). Cooldown 10s,
     tối đa 250 ký tự.
   - 2 cách vào voice: gõ ",..." hoặc lệnh /join.
   - Tự rời kênh sau 30 phút không dùng HOẶC khi kênh trống.
   - ttsApi.js gọi Vbee chế độ async rồi POLL lấy link audio (KHÔNG
     dùng webhook nữa -> nên webhookServer.js/ngrokManager.js là code chết).
   - Cache audio vào tts-cache.json (base64) để câu lặp lại đọc nhanh.

4. VẤN ĐỀ ĐÃ BIẾT / CỐ Ý ĐỂ VẬY (đừng tưởng là bug)
----------------------------------------------------------------
  - tts-cache.json (BotNoi) phình vô hạn theo thời gian (nạp hết vào
    RAM). ĐÃ chặn tạm bằng max_memory_restart + swap. Fix gốc (nếu cần)
    là giới hạn số mục cache theo LRU.
  - webhookServer.js & ngrokManager.js: code chết, không import ở đâu.
    Giữ lại cho gọn lịch sử, không ảnh hưởng gì (0 RAM, ~3KB đĩa).
  - Mật khẩu panel đang để mặc định trong code (process.env.PANEL_PASSWORD
    || '...'). Nên tách hẳn ra .env trên VPS để không lộ trong repo.
  - Điểm mặc định / điểm danh là 50 tỷ -> lạm phát, nhưng là chủ ý.
